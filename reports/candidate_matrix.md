Paths and stars are baselines for exact optimization; open LP questions concern paths' LP/root-rounding behavior and almost-star/edge-diameter-3 classes. |
| quadratic_probing | Still one of the best OpenEvolve-first candidates, but the pre-2024 'prove anything nontrivial' gap is solved: Kuszmaul-Xi prove constant expected insertion time at load factor at least 0.089 for fixed-offset schemes. Remaining work is model-specific threshold/load-factor improvement. |
| karp_rabin_collision_detection | Explicit Dagstuhl open problem with unusually concrete small-instance oracles. Downgrade evaluator fit from 5 to 4 because randomized false rejection and deterministic exact collision detection are different tasks. |
| pairing_heaps | Standard two-pass pairing heap exact decrease-key complexity remains open, but this is a hard, well-known specialist problem. Best used for potential-function/counterexample mining, not a first theorem proof. |
| list_update | Good evaluator-first candidate. The global randomized ratio gap is old and narrow; finite-state/non-projective search is useful but may not generalize. |
| unified_bound_heaps | Only promote the pointer-model version with an exact working-set definition. Broad non-pointer formulations are ambiguous because recent Dijkstra universal-optimality work already gives heaps with working-set guarantees and O(1) amortized decrease-key-like Fibonacci bounds. |
| lazy_b_trees | Promising niche theorem project. The explicit open gap is external-memory biased search trees with linear block space and weighted I/O guarantees; OpenEvolve fit is weak until a crisp finite evaluator exists. |
| succinct_compressed_structures | Dagstuhl gives explicit questions, but this folder bundles LZ indexing and grammar/DAG length-sampling. Split before promotion; recover missing notation from PDF/talk notes. |
| path_compression_topdown | Explicit Tarjan/Dagstuhl proof-simplification problem, not a new inverse-Ackermann theorem. Best as blind proof/Lean formalization after extracting the exact Seidel-Sharir recurrence. |
| splay_preorder_231 | Downgraded. Arbitrary-initial-tree Splay preorder/traversal still appears unresolved in checked sources, but the area is saturated and nearby Greedy/offline-OPT results have narrowed the useful target. Promote only a sharply defined Splay-vs-Greedy/initial-tree subproblem. |
