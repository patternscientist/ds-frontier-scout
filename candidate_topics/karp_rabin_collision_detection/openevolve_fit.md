# OpenEvolve Fit

OpenEvolve suitability: 4/5.

Plausible evaluators and artifacts:

- exact brute-force collision oracle for small strings and primes;
- generated adversarial strings/moduli;
- benchmark candidate algorithms against suffix-tree or suffix-array baselines;
- randomized false-rejection measurement.

Important caveat: this is a strong oracle for theorem-pilot support, but it is not the cleanest object-first OpenEvolve side experiment. Unless a precise algorithm grammar or adversarial string/modulus generator is chosen as the mutable object, exact checking alone should not be treated as an OpenEvolve promotion.
